Risk-sensitive Task
 
    risk-sensitive 检索任务使用和ad-hoc相同的主题，但是基于新的评估方法。risk-sensitive任务的目标是提供一个排序的网页列表，最大化返回相关文档同时最小化查询损失。所谓检索损失，我们指的是系统提供的针对特定查询低于baseline检索效率的结果。所以“risk-sesitive”任务中的“risk”指的是对于任意特定查询的我们所不希望的比基线效果差的结果。
 
    典型的，baseline针对某些查询效果很好但不是所有的，运行了一些查询后，你的系统将会有与给定baseline相关的成功或失败的分布。我们使用的risk-sensitive评估是由win/loss分布导出的，risk-sensitive任务中的评估方法将会奖励同时符合以下条件的系统：
        （a）对每一个查询的高平均效率(high average effectiveness per query)
        （b）对所给的baseline的最小损失(minimal losses with respect to the provided baseline)
        （c）对于所有查询，有高于其他系统的比例(wins affecting a higher proportion of overall queries than other systems)
 
    我们相信risk-sensitive任务会引起IR团体的兴趣因为广阔的研究领域的技术是具有实用性的，包括：健壮的查询扩展和伪相关反馈、融合和结构多样性排名、使用查询结果预测来决定对基线结果保留或者修改、优化效率和鲁棒性的learning-to-rank模型等等。TREC 2013 Web track的一个目标就是探究不同技术风险与回报权衡的实质。我们提供了样例脚本作为输入：
        （a）your risk-sensitive run
        （b）a provided baseline run
        （c）a risk parameter
